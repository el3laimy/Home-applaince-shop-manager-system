# وثيقة تصميم نظام "ALIkhlasPOS" لتجهيزات العرائس

## 1. المقدمة وهدف النظام
يهدف النظام إلى أتمتة عمليات البيع والمخازن لمحل أدوات منزلية، مع التركيز على "إدارة مجموعات المنتجات" (Bundles) ونظام "حسابات العرائس" المعقد، وتوفير تجربة بيع سريعة وموثوقة عبر نظام باركود ذكي.

## 2. الهندسة البرمجية والمكدس التقني (Tech Stack)
تم اختيار تقنيات تضمن السرعة الفائقة في البحث (Zero Latency) والموثوقية في العمليات المالية:
- **النمط المعماري**: Clean Architecture مع تقنية CQRS لفصل عمليات القراءة عن الكتابة.
- **قاعدة البيانات**: PostgreSQL (الأساسية) + Redis (للتخزين المؤقت لبيانات الباركود والأسعار).
- **واجهة المستخدم**: Flutter for Desktop (للحصول على واجهة عصرية تدعم الـ Animations واللمس).
- **الخلفية البرمجية**: .NET 9 (Web API) لضمان أعلى أداء ومعالجة متوازية للبيانات.

## 3. الوحدات الوظيفية الأساسية (Core Modules)

### أ. محرك الباركود الذكي (Smart Barcode Engine) - [محدث]
يعتبر المحرك الأساسي لسرعة العمل. يدعم النظام الآتي:
- **المعايير المدعومة**: EAN-13 للمنتجات العالمية، و Code 128 للأكواد الداخلية.
- **منطق التوليد (Generation Logic)**: نظام تلقائي لتوليد أكواد فريدة للأصناف التي لا تحمل باركود (مثل رفايع المطبخ).
- **طباعة الملصقات**: نظام متكامل للتحكم في طابعات الباركود الحرارية، يسمح بتصميم الملصق (الاسم، السعر بعد الخصم، الباركود).
- **معالجة الماسحات (Scanner Handling)**: خوارزمية تمنع تكرار السطور؛ عند مسح نفس الصنف مرتين، يتم تحديث الكمية في نفس السطر برمجيًا لتوفير المساحة وتسهيل القراءة.

### ب. إدارة المخازن والأصناف (Inventory Management)
- **الأصناف المركبة (Bundles)**: إمكانية إنشاء باركود واحد "لطقم" كامل، وعند مسحه يتم خصم جميع مكوناته الفردية من المخزن آليًا.
- **تعدد الوحدات (Multi-Unit)**: بيع الصنف بالقطعة، بالدستة، أو بالكرتونة مع ربط الأسعار تلقائياً بنسب محددة.
- **التنبيهات الذكية**: نظام تنبيه مبكر عند وصول كمية المنتجات الهامة (مثل "أطقم الحلل") إلى الحد الأدنى للطلب (مثلاً أقل من 3 أطقم).

### ج. نظام "جهاز العروسة" والمدفوعات (Bridal & Finance)
- **الحجز المسبق (Reservation)**: إمكانية حجز بضائع للعروسة وتخزينها افتراضيًا أو فعليًا في مخزن "المحجوزات" حتى اكتمال الدفع.
- **محرك الأقساط (Installments)**: جدولة الدفعات مع دعم التنبيهات عبر رسائل قصيرة أو واتساب بمواعيد الاستحقاق.
- **كشف الحساب**: استخراج تقرير مالي مفصل يوضح:
  - إجمالي المشتريات.
  - المدفوعات السابقة.
  - المبالغ المتبقية.
  - الأصناف التي لم تُستلم بعد (متبقيات الاستلام).

## 4. هيكل البيانات المقترح (Database Design)
يتم تقسيم الجداول لضمان سرعة الاستعلام وتقليل التكرار. فيما يلي الجداول الرئيسية:

| الجدول | الوظيفة | الجداول / الحقول الرئيسية |
| :--- | :--- | :--- |
| **Products** | بيانات المنتجات الأساسية | `GlobalBarcode`, `InternalBarcode`, `StockQty`, `BasePrice` |
| **Bundles** | تعريف الأطقم والمجموعات | `BundleID`, `ParentProductID`, `SubProductID`, `Discount` |
| **Invoices** | الفواتير والمبيعات | `InvoiceNo`, `CustomerID`, `TotalAmount`, `PaymentType` |
| **Installments** | إدارة تفاصيل الأقساط | `DueDate`, `Amount`, `Status (Paid/Pending)`, `ReminderSent` |

## 5. واجهة المستخدم (UI/UX)
- **شاشة البيع الرئيسية (POS Screen)**: تحتوي على مساحة كافية لعرض صور المنتجات عند المسح، مما يضمن التأكد البصري من الصنف وتجنب الأخطاء.
- **لوحة التحكم (Dashboard)**: شاشة رسومية مخصصة لصاحب المحل تظهر مؤشرات حيوية مثل: "الأصناف الأكثر مبيعًا في أجهزة العرائس" و "التدفق النقدي اليومي".
- **الوضع الليلي (Dark Mode)**: مدعوم بشكل كامل لتقليل إجهاد العين لموظفي الكاشير خلال فترات العمل الطويلة.

## 6. الأمان والموثوقية (Security)
- **النسخ الاحتياطي التلقائي (Auto-Backup)**: رفع نسخة مشفرة من قاعدة البيانات على التخزين السحابي (Cloud) كل ساعة بشكل صامت وبدون تدخل يدوي.
- **صلاحيات المستخدمين**: نظام صلاحيات دقيق يفصل بين صلاحية "تعديل السعر" وصلاحيات "إجراء البيع العادي".
- **سجل العمليات (Audit Trail)**: تعقب وتسجيل كل حركة (مسح باركود، تعديل كمية، تغيير سعر) بالوقت واسم المستخدم لمنع التلاعب المالي أو الإداري.
